# Getting Started

BrewPage is a free hosting platform for HTML pages, key-value data, JSON documents, and files. No signup required — publish content instantly.

**Base URL:** `https://brewpage.app` (alias: `https://brewdata.app` — same API)

---

## Quick Start — 4 Ways to Publish

### 1. Web UI (Zero Code)

> Open browser → Drop file → Click Publish → Get link

| Step | Action |
|:----:|--------|
| **1** | Open [brewpage.app](https://brewpage.app) |
| **2** | Drag & drop HTML / Markdown / CSV / any file |
| **3** | Click **Publish** |
| **4** | Copy your link and share |

No account needed. Supports password protection, TTL, tags, dark/light theme.

---

### 2. Claude Skill (`/brewpage`)

> Type `/brewpage` → Pick namespace → Done — link + token saved

Install the [brewcode](https://github.com/kochetkov-ma/claude-brewcode) plugin:

```bash
npx skills add kochetkov-ma/claude-brewcode
```

Then in any Claude Code session:

```
/brewpage "Hello, world!"              # text → Markdown page
/brewpage report.md                    # file → upload
/brewpage '{"status": "ok"}'           # JSON → document
/brewpage screenshot.png --ttl 1       # file, 1-day expiry
```

Or just say it naturally:

```
Publish this to brewpage
Upload report.md to brewpage.app
```

Claude detects content type, asks namespace + password, publishes, and saves the owner token to `.claude/brewpage-history.md`.

[Skill documentation](https://doc-claude.brewcode.app/brewdoc/brewpage/) · [Source](https://github.com/kochetkov-ma/claude-brewcode/tree/main/skills/brewpage-publish)

### 3. MCP Server (AI Agent)

> Tell Claude "publish this" → Claude calls BrewPage API → Get link + token

Add to your **Claude Desktop** config (`claude_desktop_config.json`):

```json
{
  "mcpServers": {
    "brewpage": {
      "command": "npx",
      "args": ["-y", "brewpage-mcp"]
    }
  }
}
```

Or **Claude Code** config (`settings.json`) — same format.

Then just ask:

| You say | What happens |
|---------|-------------|
| "Publish this HTML to BrewPage" | Creates page, returns URL |
| "Upload this image to BrewPage" | Uploads file, returns URL |
| "Delete page demo/abc123 token tok_xyz" | Deletes resource |

[MCP Server guide](/brewpage-openapi/mcp-server/) · [Source](https://github.com/kochetkov-ma/brewpage-openapi/tree/main/mcp-server)

### 4. REST API (curl / code)

> POST JSON → Get URL + owner token → Share

**Publish HTML:**

```bash
curl -X POST https://brewpage.app/api/html \
  -H "Content-Type: application/json" \
  -d '{"content":"<h1>Hello World</h1>","format":"HTML"}'
```

**Publish Markdown:**

```bash
curl -X POST https://brewpage.app/api/html \
  -H "Content-Type: application/json" \
  -d '{"content":"# My Notes\n\nMarkdown is **auto-rendered**.","format":"MARKDOWN"}'
```

**Upload a file:**

```bash
curl -X POST https://brewpage.app/api/files/uploads \
  -F "namespace=assets" \
  -F "file=@image.png"
```

**Response:**

```json
{
  "namespace": "demo",
  "id": "aBcDeFgHiJ",
  "url": "https://brewpage.app/demo/aBcDeFgHiJ",
  "shortUrl": "https://brewpage.app/s/xYz123",
  "ownerToken": "tok_abc123..."
}
```

> **Save your `ownerToken`!** You need it to update or delete content later. It cannot be recovered.

## More API Examples

**Store key-value data:**

```bash
curl -X PUT https://brewpage.app/api/kv/myapp/config/theme \
  -H "Content-Type: application/json" \
  -d '{"value": "dark"}'

curl https://brewpage.app/api/kv/myapp/config/theme
```

**Store JSON document:**

```bash
curl -X POST https://brewpage.app/api/json/myapp/users \
  -H "Content-Type: application/json" \
  -d '{"document":{"name":"Alice","role":"admin"}}'
```

**Platform stats:**

```bash
curl https://brewpage.app/api/stats
```

**With options (password + TTL + tags):**

```bash
curl -X POST https://brewpage.app/api/html \
  -H "Content-Type: application/json" \
  -d '{
    "namespace": "myproject",
    "content": "# Private Notes",
    "format": "MARKDOWN",
    "password": "secret123",
    "ttlDays": 90,
    "tags": ["notes"]
  }'
```
